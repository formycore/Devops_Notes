<p>Hello everyone and welcome to Day 1 of 21 Days of AWS using Terraform, the topic for today is Introduction to Terraform.  Let&#8217;s begin our AWS automation journey using Terraform. </p>



<blockquote class="wp-block-quote"><p><strong>What is terraform?</strong></p></blockquote>



<p><em>Terraform is a tool for provisioning infrastructure(or managing Infrastructure as Code). It supports multiple providers(eg, AWS, Google Cloud, Azure, OpenStack..)</em>.</p>



<p><a href="https://www.terraform.io/docs/providers/index.html">https://www.terraform.io/docs/providers/index.html</a></p>



<blockquote class="wp-block-quote"><p><strong><em>Installing Terraform</em></strong></p></blockquote>



<p><em>Installing Terraform is pretty straightforward, download it from Terraform download page and select the appropriate package based on your operating system</em>.</p>



<p><a href="https://learn.hashicorp.com/terraform/getting-started/install.html">https://learn.hashicorp.com/terraform/getting-started/install.html</a></p>



<p><a href="https://www.terraform.io/downloads.html">https://www.terraform.io/downloads.html</a></p>

<p><em>As mentioned above terraform support many providers, for my use case I am using AWS.</em></p>



<pre class="wp-block-preformatted"><strong><em>Prerequisites</em></strong><em>1: Existing AWS Account(OR Setup a new account)<br>2: IAM full access(OR at least have AmazonEC2FullAccess)<br>3: AWS Credentials(AWS_ACCESS_KEY_ID and AWS_SECRET_ACCESS_KEY)</em></pre>



<p><em>Once you have pre-requisites 1 and 2 done, the first step is to export Keys AWS_ACCESS_KEY_ID and AWS_SECRET_ACCESS_KEY.</em></p>



<pre class="wp-block-code"><code>export AWS_ACCESS_KEY_ID="your access key id here"
export AWS_SECRET_ACCESS_KEY="your secret access key id here"</code></pre>



<p><strong><em>NOTE:&nbsp;</em></strong><em>These two variables are bound to your current shell, in case of reboot, or if open a new shell window, these changes will be lost</em></p>



<p><em>With all pre-requisites in place, it’s time to write your first terraform code, but before that just a brief overview about terraform language</em></p>



<ul><li><em>Terraform code is written in the HashiCorp Configuration Language(HCL)</em></li><li><em>All the code ends with the extension of .tf</em></li><li><em>It’s a declarative language(We need to define what infrastructure we want and terraform will figure out how to create it)</em></li></ul>



<p><em>In this first example I am going to build EC2 instance, but before creating EC2 instance go to AWS console and think what the different things we need to build EC2 instance are</em></p>



<ul><li><em>Amazon Machine Image(AMI)</em></li><li><em>Instance Type</em></li><li><em>Network Information(VPC/Subnet)</em></li><li><em>Tags</em></li><li><em>Security Group</em></li><li><em>Key Pair</em></li></ul>



<p><em>Let break each of these steps by step</em></p>



<ul><li><strong><em>Amazon Machine Image(AMI):</em></strong><em>&nbsp;It’s an Operating System Image used to run EC2 instance. For this example, I am using Centos 7  </em>ami-01ed306a12b7d1c96<em>. We can create our own AMI using AWS console or Packer.</em></li></ul>



<p><a href="https://www.packer.io/intro/">https://www.packer.io/intro/</a></p>

<ul><li><strong><em>Instance Type:</em></strong><em>&nbsp;Type of EC2 instance to run, as every instance type provides different capabilities(CPU, Memory, I/O). For this example, I am using t2.micro(1 Virtual CPU, 1GB Memory)</em></li></ul>

<pre class="wp-block-preformatted"><em>git add main.tf</em></pre>



<pre class="wp-block-preformatted"><em>git commit -m "first terraform EC2 instance"</em></pre>

<pre class="wp-block-preformatted"><em>git add .gitignore</em></pre>



<pre class="wp-block-preformatted"><em>git commit -m "Adding gitignore file for terraform repository"</em></pre>

<pre class="wp-block-preformatted"><em>$ cat .gitignore</em></pre>

<pre class="wp-block-preformatted"><em>git remote add origin https://github.com/&lt;user name&gt;/terraform.git</em></pre>



<ul><li><em>Push the code</em></li></ul>



<pre class="wp-block-preformatted"><em>$ git push -u origin master</em></pre>
